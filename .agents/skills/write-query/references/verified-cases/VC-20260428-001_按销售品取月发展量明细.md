# 按销售品取月发展量明细

## 适用

- 用户要按销售品编码/名称取发展量或发展明细。
- 需求包含揽装人、揽装机构、划小局向、是否竣工等字段。

## 不适用

- 用户要销售品存量/在档，此时主表应为 014 优惠资料表。
- 用户明确只看某个专项产品清单。

## 主表与口径

- 主表：041 优惠订单表 `dwm_yz_rpt_comm_ba_msdisc_final`。
- 发展量动作：`action_id IN (1292, 6200)`。
- 动作类统计排除撤单作废：`COALESCE(subs_stat_reason,'-1') NOT IN ('1200','1300')`。
- 是否竣工默认输出标记列：`CASE WHEN subs_stat='301200' THEN 1 ELSE 0 END AS is_jg`。

## 字段映射

| 需求字段 | 来源 |
|----------|------|
| 订单 / 号码 | 041 `subs_id`、`subs_code`、`serv_id`、`acc_nbr` |
| 销售品 ID | 041 `prod_offer_id` |
| 销售品编码 / 名称 | 020 销售品维表 |
| 揽装人 | 041 `sales_code`、`sales_man_name`，缺失再补销售员表 |
| 揽装分局 / 营服 | 041 `salestaff_subst_id`、`salestaff_branch_id` 补机构维表 |
| 划小局向 | 069 按 `serv_id` 补 `subst_name`、`branch_name` |

## 补表

- 销售品维表：`dws_crm_cfguse.dws_offer`，必须 `city_id=200`。
- 机构维表：`dwd_yz_dim_org`，分局 `levs=3`，营服 `levs=4`。
- 全业务资料表：仅当 041 缺划小局向名称时，按 `serv_id` 补 069。

## SQL 编排

- 先在 041 过滤动作、状态原因和时间。
- 再补销售品、机构、必要的 069 字段。
- 明细需求不要混入行级汇总窗口；汇总需求单独 `GROUP BY`。
