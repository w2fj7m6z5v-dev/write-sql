# 按营服汇总主宽与 129+ 宽带入网

## 适用

- 用户要按县分、营服、月份汇总主宽、129+ 宽带等宽带入网指标。
- 用户没有明确指定专项新装表，或希望按全业务资料表已有归属字段汇总。

## 不适用

- 用户要求宽带专项新装清单独有字段。
- 用户指定以某个标准指标文件的 SQL 为唯一口径。

## 主表与口径

- 主表优先 069 全业务资料表：`tables/069_全业务资料表.md`。
- 标准指标命中时，先读 `METRIC_INDEX.md` 中的主宽入网、129+ 宽带入网相关指标，再打开单指标文件。
- 与指标 SQL 冲突时，以单指标技术口径为准。

## 字段映射

| 需求字段 | 来源 |
|----------|------|
| 月份 | 069 `par_month_id` |
| 县分 / 分局 | 069 `subst_name` 或指标 SQL 指定字段 |
| 营服 | 069 `branch_name` 或指标 SQL 指定字段 |
| 主宽 / 129+ 宽带数量 | 指标文件或 069 常用条件 |

## 补表

- 069 已有县分、营服名称时不补机构维表。
- 只有用户要求订单受理、竣工、协销等订单事实字段时，才补 040 全业务号码订单表。

## SQL 编排

- 汇总类需求用 `GROUP BY par_month_id, subst_name, branch_name`。
- 多个指标可用条件聚合；标准指标明确 SQL 时优先按指标 SQL 改写。
