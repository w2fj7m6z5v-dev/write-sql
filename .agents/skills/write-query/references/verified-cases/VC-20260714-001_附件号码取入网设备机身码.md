# VC-20260714-001：附件号码取入网时光猫设备机身码（订单维度）

## 适用问题

附件号码清单需要取入网时绑定的光猫设备机身码，核心特征：
- 需要取**入网时**的设备，而非当前在用设备
- 通过**入网订单**（`subs_id`）关联设备，而非通过号码（`serv_id`）关联
- 区别于 119 号码设备关系表（号码维度，当前状态）

## 不适用

- 取当前在用设备信息（用 119 号码设备关系表）
- 非宽带产品的入网设备
- 不需要设备信息，只需入网订单信息（只走步骤1即可）

## 核心思路

### 关键区分：号码设备表 vs 订单设备表

| 维度 | 119 号码设备关系表 | 144 CRM设备订单关系表 |
|------|-------------------|---------------------|
| 关联键 | `serv_id`（号码） | `order_item_id`（订单ID） |
| 含义 | 号码当前绑定的设备 | 订单办理时绑定的设备 |
| 时效 | 当前快照 | 入网时设备 |
| 适用 | 查当前在用什么设备 | 查入网时用什么设备 |

### 编排步骤

```
步骤1：附件号码 → 069 月表取入网订单信息
       (serv_id, subs_id, subs_code, open_date, act_date, subs_stat_date)
步骤2：subs_id → 144 CRM设备订单关系表 → eqpt_sn（机身码）
       归档表(_his) + 当前表 UNION ALL，status_cd='1100'
```

### 069 月表自带入网订单字段

069 全业务资料表已包含入网订单相关字段，无需回查 040 订单表：

| 069 字段 | 含义 | 对应 040 字段 |
|----------|------|-------------|
| `subs_id` | 入网订单标识 | `subs_id` |
| `subs_code` | 入网订单编码 | `subs_code` |
| `act_date` | 入网受理日期 | `act_date` |
| `subs_stat_date` | 入网订单状态日期 | `subs_stat_date` |
| `open_date` | 服务开通时间 | - |

## SQL 编排模板

### 步骤1：附件号码 → 069 月表取入网订单信息

```sql
drop table if exists tmp_{scope}_install_order purge;
create table tmp_{scope}_install_order
row format delimited fields terminated by '\u0001'
stored as orc tblproperties('orc.compression'='snappy') as
select
    a.acc_nbr,
    b.serv_id,
    b.subs_id,
    b.subs_code,
    b.open_date,
    b.act_date,
    b.subs_stat_date,
    b.subst_name,
    b.branch_name,
    b.state,
    b.is_cancel_user,
    b.hist_create_date,
    b.par_month_id as install_month
from (
    -- 附件种子表
    select acc_nbr from tmp_{scope}_seed_raw
) a
left join (
    select
        serv_id, acc_nbr, subs_id, subs_code,
        open_date, act_date, subs_stat_date,
        subst_name, branch_name, state,
        is_cancel_user, hist_create_date,
        par_month_id,
        row_number() over (partition by acc_nbr order by par_month_id asc) as rn
    from dwm_yz_tb_comm_cm_all_mon_final
    where is_new_user = 1
      and prod_type = 40
      and kd_desc = '普通宽带'
) b
  on a.acc_nbr = b.acc_nbr and b.rn = 1;
```

### 步骤2：入网订单 → CRM设备订单表取机身码

```sql
drop table if exists tmp_{scope}_install_device purge;
create table tmp_{scope}_install_device
row format delimited fields terminated by '\u0001'
stored as orc tblproperties('orc.compression'='snappy') as
select
    o.acc_nbr,
    o.serv_id,
    o.subs_id,
    o.subs_code,
    o.open_date,
    o.act_date,
    o.subs_stat_date,
    o.subst_name,
    o.branch_name,
    o.state,
    o.is_cancel_user,
    o.hist_create_date,
    o.install_month,
    r.eqpt_sn
from tmp_{scope}_install_order o
left join (
    select prod_inst_id, order_item_id, eqpt_sn
    from (
        select prod_inst_id, order_item_id, eqpt_sn
        from dws_crm_order.dws_ord_prod_res_inst_rel_his
        where city_id = 200 and status_cd = '1100'
        union all
        select prod_inst_id, order_item_id, eqpt_sn
        from dws_crm_order.dws_ord_prod_res_inst_rel
        where city_id = 200 and status_cd = '1100'
    ) t
    group by prod_inst_id, order_item_id, eqpt_sn
) r
  on o.subs_id = r.order_item_id;
```

## 自检要点

1. **步骤1命中率**：附件号码在069月表中能匹配到入网记录的比例
2. **步骤2设备命中率**：有入网订单的号码中，能匹配到设备机身码的比例
3. **同一订单多设备**：一个入网订单可能绑定多台设备，当前保留明细
4. **归档表覆盖**：`_his` 归档表是否覆盖所有历史入网订单

## 风险

- **`subs_id` 类型匹配**：069 的 `subs_id` 为 `decimal(22,0)`，CRM 设备订单表的 `order_item_id` 类型需确认是否一致，如不一致需 `cast`
- **已拆号码的入网月**：`is_new_user=1` 只出现在入网当月分区，必须扫月表而非当月快照
- **月表覆盖范围**：需确认月表保留了多少历史月，早期入网号码可能匹配不上
