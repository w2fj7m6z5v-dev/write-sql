# 台阶收入多维度按月打横

## 适用

- 用户要台阶收入（M-TOPIC-REV-004），按营服/增存类型/付费类型/业务产品类型/套餐价值档次/收入来源等维度分组，并按月份打横 pivot。
- 主表为 101 台阶收入清单，无需补表，单步 CTAS 即可完成。

## 不适用

- 不是台阶收入口径（应走对应收入表/指标）。
- 只需单月汇总不需打横。
- 需要补充营服/机构等维度中文名称（表已自带）。

## 主表与口径

- 主表：`ads_yz_xsb_tjsr_skj_list_db`（台阶收入清单，101）。
- 度量：`SUM(fee_all)` 税后收入。
- 分区：`par_month_id`。
- 标准指标：M-TOPIC-REV-004。

## 字段映射

| 需求维度 | 来源字段 | 注意 |
|----------|----------|------|
| 营服名称 | `branch_name` | 划小营服，非揽装营服 |
| 增存类型 | `zc_type` | 表自带 |
| 付费类型 | `payment_type` | 表自带 |
| 业务产品类型 | `prod_type_yw` | 产品类型打标，非 `prod_type` |
| 套餐价值档次 | `rh_tc_value` → CASE WHEN 分段 | 见 `dictionaries/rh_tc_value.md` |
| 收入来源 | `data_src_name` | 表自带 |

## 补表

无需补表。101 表已自带全部维度字段。

## SQL 编排

- 单步 CTAS：`DROP + CREATE TABLE ... AS SELECT`。
- 月份打横：`SUM(CASE WHEN par_month_id = YYYYMM THEN fee_all ELSE 0 END) AS m_YYYYMM`。
- CASE WHEN 分段表达式需在 SELECT 和 GROUP BY 中重复（Hive 不支持 GROUP BY 别名）。
- 直接用 `par_month_id BETWEEN start AND end` 裁剪分区，不额外过滤业务时间。

## 参考 SQL

```sql
drop table if exists tmp_tjsr_xxx_pivot purge;
create table tmp_tjsr_xxx_pivot
row format delimited fields terminated by '\u0001'
stored as orc tblproperties('orc.compression'='snappy') as
select
    branch_name,
    zc_type,
    payment_type,
    prod_type_yw,
    case
        when rh_tc_value < 0   then '(-∞,0)'
        when rh_tc_value >= 0  and rh_tc_value < 59  then '[0,59)'
        when rh_tc_value >= 59 and rh_tc_value < 79  then '[59,79)'
        when rh_tc_value >= 79 and rh_tc_value < 129 then '[79,129)'
        when rh_tc_value >= 129 and rh_tc_value < 199 then '[129,199)'
        when rh_tc_value >= 199 and rh_tc_value < 299 then '[199,299)'
        when rh_tc_value >= 299 and rh_tc_value < 399 then '[299,399)'
        when rh_tc_value >= 399 and rh_tc_value < 499 then '[399,499)'
        when rh_tc_value >= 499 and rh_tc_value < 599 then '[499,599)'
        when rh_tc_value >= 599 and rh_tc_value < 699 then '[599,699)'
        when rh_tc_value >= 699 then '[699,+∞)'
        else '其他'
    end as rh_value_type,
    data_src_name,
    sum(case when par_month_id = ${start_month_1} then fee_all else 0 end) as m_${start_month_1},
    -- ... 逐月展开 ...
    sum(case when par_month_id = ${end_month} then fee_all else 0 end) as m_${end_month}
from ads_yz_xsb_tjsr_skj_list_db
where par_month_id between ${start_month} and ${end_month}
    and branch_name = '${branch_name}'
group by
    branch_name,
    zc_type,
    payment_type,
    prod_type_yw,
    case
        when rh_tc_value < 0   then '(-∞,0)'
        when rh_tc_value >= 0  and rh_tc_value < 59  then '[0,59)'
        when rh_tc_value >= 59 and rh_tc_value < 79  then '[59,79)'
        when rh_tc_value >= 79 and rh_tc_value < 129 then '[79,129)'
        when rh_tc_value >= 129 and rh_tc_value < 199 then '[129,199)'
        when rh_tc_value >= 199 and rh_tc_value < 299 then '[199,299)'
        when rh_tc_value >= 299 and rh_tc_value < 399 then '[299,399)'
        when rh_tc_value >= 399 and rh_tc_value < 499 then '[399,499)'
        when rh_tc_value >= 499 and rh_tc_value < 599 then '[499,599)'
        when rh_tc_value >= 599 and rh_tc_value < 699 then '[599,699)'
        when rh_tc_value >= 699 then '[699,+∞)'
        else '其他'
    end,
    data_src_name;
```

## 自检

```sql
-- 1. 确认分区数据量级
select par_month_id, count(*) as cnt, sum(fee_all) as total_fee
from ads_yz_xsb_tjsr_skj_list_db
where par_month_id between ${start_month} and ${end_month}
    and branch_name = '${branch_name}'
group by par_month_id order by par_month_id;

-- 2. 结果表量级
select count(*) from tmp_tjsr_xxx_pivot;

-- 3. 结果表样例
select * from tmp_tjsr_xxx_pivot limit 10;
```
