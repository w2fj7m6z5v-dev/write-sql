# 业务场景 → 候选表 路由表

> **用途**：SKILL 第二步「锁主表 / 经验路由选表」必读。主表未知时，先查这张表拿候选表，再去 A 层加载具体表 md。
>
> **三列要点**：
> - **优先级 1**：默认走这张
> - **优先级 2 / 备选**：特定客群或字段不全时改走
> - **不要选**：典型陷阱，agent 易因字段名匹配选错
>
> **回填规则**：每次有用户纠正"应该走 X 而不是 Y"，立刻把正反两条都加进来；新场景在表末追加。

## 高优先级覆盖规则

- 用户已明确主表、字段规则或生产经验时，以用户规则覆盖本表默认优先级。
- 常规业务需求里，移动/宽带新装、到达、拆机这类全业务事实，优先先看 069 全业务资料表。
- 069 全业务资料表分日表/月表：近半年账期优先日表 `dwm_yz_tb_comm_cm_all_final`，更早历史账期走月表 `dwm_yz_tb_comm_cm_all_mon_final`；重叠账期默认优先日表。
- 若用户要订单编码、订单状态、受理时间、协销人等订单事实字段，再补 040 全业务号码订单表；040 仍不足时，再看 042/043 协销表。
- 专项新装清单只在用户明确点名专项表，或 069 字段明确不足时再启用。

## 选表决策三问（写 SQL 前必答）

1. **业务范围**：用户的"销售品/新装/收入"是指**任意范围**还是**某专项产品**？
   → 任意 → 走通用主表；专项 → 走专项清单
2. **是动作还是存量**？
   → 动作（发展、订购、续约、拆机）→ 走订单/动作表；存量（在档、生效）→ 走资料表
3. **要明细字段还是只算数**？
   → 算数 → 资料表标志位即可；明细 → 专项清单字段更全
