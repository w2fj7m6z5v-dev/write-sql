# Schema Linking / 主表路由

用于主表未知时，把用户业务语言映射到 CDAP 主事实表。Schema Linking 阶段**默认只读本文件**；生产表名查找用 `TABLE_INDEX.md`，标准指标用 `METRIC_INDEX.md`。

## 本文件回答什么

| 问题 | 本文件怎么回答 |
|------|----------------|
| 用户这个需求应该从哪张主表出发？ | 看“业务对象 + 动作/状态 + 输出粒度”命中主表路由表 |
| 哪些表只是备选或补表？ | 看“备选 / 补表”列，不要直接改主表 |
| 哪些表容易被字段名误导误选？ | 看“不要选”列 |
| 路由不确定怎么办？ | 输出 1 张拟用主表 + 备选表 + 待确认项，不直接写 SQL |
| 专项场景 JOIN/步骤/审计细节？ | **不读本文件展开**；命中后读 `scenarios/SC-*.md` |

## 快速使用

1. 先看用户是否已指定主表；指定则不要泛化路由。
2. 若命中标准指标，先到 `METRIC_INDEX.md` 找指标，再用指标文件约束主表。
3. 若不是标准指标，用“业务对象 + 动作/状态 + 输出字段”查本文件。
4. 若命中附件驱动、跨表编排或专项诊断，先在 `scenarios/INDEX.md` 找命中场景；本文件只保留主表/事实表判断。
5. 输出 1 张拟用主表、最多 2 张备选表，并列出“不选哪些表”。

## 专项场景索引

专项场景的唯一索引是 `scenarios/INDEX.md`。路由阶段只判断主事实表和不要误选的表；字段映射、补表步骤、过程表血缘、风险审计与自检不在本文件重复展开，命中后先读 `scenarios/INDEX.md`，再只打开对应 `SC-*.md`。

## 路由输入

做主表选择前，必须先从需求里抽出这些字段。缺项不一定要追问，但要在“主表确认”的风险/备注里标出来。

| 输入项 | 示例 | 影响 |
|--------|------|------|
| 业务对象 | 销售品、移动、宽带、收入、积分、续约 | 决定候选表域 |
| 动作/状态 | 发展、订购、在档、到达、出账、拆机 | 决定动作表还是存量表 |
| 输出粒度 | 明细、按月汇总、按营服汇总 | 决定事实表粒度是否适配 |
| 输出字段 | 订单编码、销售品名、协销人、收入金额 | 决定是否补表，不直接决定主表 |
| 客群/产品范围 | 公众、商企、小微、FTTR、燃气卫士 | 决定通用表还是专项表 |
| 标准指标名 | 主宽到达数、净增积分、台阶收入 | 命中时优先读 `METRIC_INDEX.md` |

## 路由决策树

```text
用户已指定主表？
  是 -> 采用用户主表，进入字段映射
  否 -> 是否命中标准指标？
      是 -> METRIC_INDEX + 单指标文件约束主表
      否 -> 判断业务事实类型
          动作事实：订单/续约/拆机/积分/收入专项表
          入网/新装/到达规模：069 全业务资料表
          存量状态：069 全业务资料表或资料类存量表
          维度名称：先看主表字段，不足再补维表
```

## 置信度

主表确认时给出置信度，帮助用户判断是否需要停下来纠偏。

| 置信度 | 条件 | 处理 |
|--------|------|------|
| 高 | 用户指定主表；或标准指标明确技术 SQL；或路由表唯一命中 | 输出主表确认，等待确认后继续 |
| 中 | 路由表命中，但输出字段可能依赖补表 | 输出主表 + 补表风险，等待确认 |
| 低 | 业务对象/动作不清，或多个主表都合理 | 不写 SQL；列候选表和需要用户确认的问题 |

## 覆盖规则

- 用户已明确主表、字段规则或生产经验时，以用户规则为准。
- 标准指标命中时，以 `METRIC_INDEX.md` 和单指标技术口径约束主表。
- 任意产品入网量、新装量、到达量，以及常规在网、出账、拆机等全业务规模类需求，优先看 069 全业务资料表。
- 069 日/月表：区间全在当前表最近 4 个月窗口内用日表；否则月表能覆盖整段查询账期则全程月表；仅月表缺最新月时用日表补缺口 UNION。详见 `tables/069_全业务资料表.md`。
- **历史某月快照 / 拆机前 N 月属性回溯**：069 与特性资料表均走 **月表**（069 `dwm_yz_tb_comm_cm_all_mon_final`、特性 `tb_pre_cm_attr_all_mon`），账期口径保持一致；不要用 069 日表配特性月表。
- **拆机默认口径**：未特别声明「物理拆机」时，默认 **逻辑拆机**（069 `is_cancel_user=1` + `hist_create_date`）；物理拆机（`is_wl_cancel_user=1` + `wl_cancel_subs_stat_date`）需用户特说明。逻辑拆机不一定有拆机订单。
- 069 拆机标签按账期快照：**仅拆机当月** `par_month_id` 分区下才有 `is_cancel_user=1`；反查拆机月时 `par_month_id` 即拆机月。
- 拆机号码相关信息（如主套餐、客户/机构、产品属性、特性、融合等）默认取**拆机前一个月**快照；例如 5 月拆机，相关信息匹配 4 月。拆机当月分区只负责识别拆机事实，不默认代表拆机前状态。
- 用户说「属性 / attr_id / 特性值」时：上下文有 **附属产品 / `sub_prod_id`** → 106；否则默认 **105 产品规格属性**。
- 用户说「特性名称 / attr_name / 根据产品规格编码或附属产品编码查特性 ID」时：补 **016 特性规格维表**；用户说「特性值中文名」时补 **015 特性值字典**，二者不要混用。
- 用户说「IMSI / 号码 IMSI / 根据号码导 IMSI」时，主路径 069 → 105；完整步骤见 **§专项场景索引**（SC-005）。
- 要订单编码、订单状态、受理时间、协销人等订单事实字段时，先补 040 全业务号码订单表；040 不足再看协销专表。040/041 月表的 `par_month_id` 是归档月，**当前表没有 `par_month_id`**；按 `act_date/subs_stat_date` 查历史业务时间订单时，应先合并足够范围的归档月表与当前表，再按业务时间字段过滤。
- 专项产品表只在用户明确点名专项，或通用主表字段明确不足时使用。

## 时间字段语义

`state` 之外，时间字段也是高频被误用的对象。「分区是读哪些，业务时间字段是算哪些」——分清职责。

| 用户说 | 字段 | 适用表 | 说明 |
|--------|------|--------|------|
| 受理时间 | `act_date` | 040/041/022 等订单表 | 订单受理时点；040/041 常见格式 `yyyy-MM-dd HH:mm:ss.S`，按自然日过滤用 `substr(cast(act_date as string),1,10)` 或等价日期转换 |
| 竣工时间 | `subs_stat_date`（配 `subs_stat='301200'`） | 040/041/022 等订单表 | 订单状态变为竣工的日期；常见格式 `yyyy-MM-dd HH:mm:ss.S`，按自然日过滤用 `substr(cast(subs_stat_date as string),1,10)` 或等价日期转换 |
| 主宽入网按竣工 / 主宽按竣工时间 | `open_date` | 069 全业务资料表；M-BASIC-BB-001/002 | 业务口语「竣工」在主宽入网上**统一映射 `open_date`**，不必追问；≠ 订单表 `subs_stat_date` |
| 开通时间 / 入网时间 | `open_date` | 069 + 部分订单表 | 服务开通时点；069 入网类默认时间字段 |
| 拆机时间（默认） | `hist_create_date` | 069（逻辑拆机） | 未特说明物理拆机时用此字段 |
| 物理拆机时间 | `wl_cancel_subs_stat_date` | 069 / 拆机清单 | 用户明确物理拆机时用 |
| 携入 / 携出时间 | `xr_date` / `xc_date` | 069 | 携号转网相关 |
| 分区裁剪 | `par_month_id` / `par_day_id` | 各表 frontmatter `partition_keys` | 仅用于 IO 裁剪，不替代业务时间过滤 |
| 拍照月 / 快照月 | `par_month_id`（上年12月） | 069 月表 `dwm_yz_tb_comm_cm_all_mon_final` | 年度基准对比月，通常为上年12月（如当前2026年取 `202512`）；净增 = 当前月到达 − 拍照月到达；≠ 去年同期 |

## 找不到路由时

如果路由表没有命中：

1. 用关键词在 `TABLE_INDEX.md` 查业务主题、表名、Hive 表名。
2. 若像标准指标，先用 `METRIC_INDEX.md` 查指标名或近似指标。
3. 若只找到候选表，不要写 SQL；先输出“低置信度主表确认”。
4. 用户确认后，再打开命中 `tables/*.md` 盘字段。
