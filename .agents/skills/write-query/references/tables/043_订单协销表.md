# 订单协销表

- **Hive 表名**: `zone_gz_yz.dwd_yz_ba_obj_xx_final`
- **视图名**: view_dwd_yz_ba_obj_xx_final
- **业务负责人**: 
- **业支负责人**: 杨洋
- **粒度**: 订单项 + 发展人类型粒度
- **使用规则**: 本表没有月表；历史或当期均使用当前表，并按 `par_month_id` 过滤

---

## 字段说明

| 字段 | 字段类型 | 字段含义 |
|------|---------|---------|
| order_item_id |  | 订单项标识；常用主表 `subs_id` 关联 |
| par_month_id |  | 账期 / 统计月份 |
| dev_staff_type |  | 发展人类型 |
| dev_staff_id |  | 发展人ID |
| his_create_date |  | 归档时间 |
| his_month_id |  | 归档月份 |
| staff_name |  | 发展人名称 |
| sales_code |  | 发展人揽装工号 |

## 发展人类型口径

| dev_staff_type | 口语含义 | 输出字段建议 |
|---|---|---|
| `2000` | 第一协销人 / 第二发展人 | `dev_staff_id/staff_name/sales_code` as 第一协销人 |
| `3000` | 第二协销人 / 第三发展人 | `dev_staff_id/staff_name/sales_code` as 第二协销人 |

## 常用链路

### 订单补第一/第二协销人

```sql
select
  a.*,
  b.dev_staff_id as xx_salestaff_id1,
  b.staff_name as xx_salestaff_name1,
  b.sales_code as xx_salestaff_code1,
  c.dev_staff_id as xx_salestaff_id2,
  c.staff_name as xx_salestaff_name2,
  c.sales_code as xx_salestaff_code2
from tmp_order a
left join (
  select *
  from zone_gz_yz.dwd_yz_ba_obj_xx_final
  where dev_staff_type = '2000'
    and par_month_id = ${sum_month}
) b
  on a.subs_id = b.order_item_id
left join (
  select *
  from zone_gz_yz.dwd_yz_ba_obj_xx_final
  where dev_staff_type = '3000'
    and par_month_id = ${sum_month}
) c
  on a.subs_id = c.order_item_id;
```

## JOIN 与风险

- JOIN 键：`subs_id = order_item_id`。
- 必须按 `par_month_id` 过滤或与主表账期对齐。
- 本表没有 `_mon_final` 月表，不要按 042 的历史月表规则套用。
- 若同一订单同一 `dev_staff_type` 多条记录，先按业务时间或去重规则收敛后再 JOIN，避免放大订单明细。
