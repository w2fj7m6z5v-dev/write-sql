# 产品实例账户关系表

- **Hive 表名 / 生产现网名**：`dws_crm_cust.dws_prod_inst_acct_rel_aap`
- **用途**：按服务实例 / 产品实例回填缴费账户 `acct_id`，作为继续补支付方案、外部账户、银行信息的入口。
- **典型入口**：用户提供 `serv_id` / `prod_inst_id` 清单，或附件只有号码时先用 069 补 `serv_id`。
- **默认过滤**：广州固定 `city_id=200`。

## 字段说明

| 字段 | 字段类型 | 字段含义 | 说明 |
| ------ | ------ | ---------- | ------ |
| prod_inst_id | string | 产品实例标识 | 通常与主表 `serv_id` 关联 |
| acct_id | string | 缴费账户标识 | 后续关联 132 支付方案表 |
| city_id | string | 地市 | 广州固定 `city_id=200` |

## 常用链路

```sql
select
  seed.*,
  rel.acct_id
from tmp_seed seed
left join dws_crm_cust.dws_prod_inst_acct_rel_aap rel
  on cast(seed.serv_id as decimal(22,0)) = cast(rel.prod_inst_id as decimal(22,0))
 and rel.city_id = 200;
```

## JOIN 与风险

- JOIN 键：`serv_id = prod_inst_id`，建议两侧统一转 `decimal(22,0)` 或同一字符类型。
- 同一 `prod_inst_id` 可能存在多个账户关系；默认保留明细。若要一服务一行，需确认按状态、时间或主账户口径去重。
- 本表只提供 `acct_id`，不提供银行账户名称或银行名称；账户名称继续补 132/133，银行名称继续补 134。
