# 附属产品配置表

- **Hive 表名**：`dwd_dim_all_config`
- **用途**：按附属产品名称、配置类型等查询附属产品 ID / 编码 / 名称。

---

## 字段说明

| 字段 | 字段类型 | 字段含义 | 说明 |
| ------ | ------ | --------- | ------ |
| seq_id | int | 配置序号 | 附属产品配置常见 `seq_id=12` |
| seq_type | string | 配置类型 | 附属产品 ID 常见 `seq_type='sub_prod_id'` |
| seq_name | string | 配置名称 | 如附属产品名称 |
| seq_value_id | int | 配置值 ID | 可关联 106.`sub_prod_id` |
| seq_value_code | string | 配置值编码 | 可作为附属产品编码输出 |
| reamark | string | 配置备注 | 原字段拼写按生产表保留 |

---

## 常用 JOIN 模式

按附属产品名称圈定 106 附属产品资料：

```sql
select
  a.serv_id,
  a.sub_prod_id,
  a.attr_id,
  a.attr_value1,
  b.seq_name as sub_prod_name,
  b.seq_value_code as sub_prod_code
from summary_ods_day_city.rpt_comm_cm_subserv a
join dwd_dim_all_config b
  on a.sub_prod_id = b.seq_value_id
 and b.seq_id = 12
 and b.seq_type = 'sub_prod_id'
where a.par_corp_id = '200'
```

---

## 注意事项

- 本表只做附属产品配置翻译或筛选，不替代 106 附属产品资料表。
- 具体 `seq_name`、`seq_value_code`、产品编码清单来自需求或配置查询结果，不在技能中硬编码。
- 与 106 JOIN 后，仍以 106 的 `serv_id + attr_id/sub_prod_id` 明细粒度为准；一服务多附属产品时默认保留明细。
