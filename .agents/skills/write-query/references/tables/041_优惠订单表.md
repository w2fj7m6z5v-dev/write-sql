# 优惠订单表

- **Hive 表名（md 历史）**: `zone_gz_yz.dwm_yz_rpt_comm_ba_msdisc_final`
- **Hive 表名（生产现网 / 当前表）**: `dwm_yz_rpt_comm_ba_msdisc_final` ← **落 SQL 时使用此名（无 schema 前缀）**
- **月表 / 历史归档表名**: `dwm_yz_rpt_comm_ba_msdisc_mon_final`
- **视图名**: `view_dwm_yz_rpt_comm_ba_msdisc_final`
- **业务负责人**: 
- **业支负责人**: 杨洋

> ⚠️ **表名漂移**：md 历史名带 `zone_gz_yz.` 前缀，生产现网实际无前缀。落 SQL 前请向用户校对（参考 [RULES.md「选表与字段」](../RULES.md)）。

> ⚠️ **当前表 vs 历史月表**：订单生命周期为“受理 → 竣工 → 归档”。月表 `dwm_yz_rpt_comm_ba_msdisc_mon_final` 只沉淀已归档优惠订单，`par_month_id` 表示**归档月**，不是受理月，也不严格等于竣工月；例如 `par_month_id=202605` 表示 2026 年 5 月归档的优惠订单，其中既可能有 5 月受理/竣工/归档订单，也可能有历史受理或历史竣工、5 月才归档的订单。
>
> 当前表 `dwm_yz_rpt_comm_ba_msdisc_final` 是当前订单池，包含当前月归档订单，以及历史受理/历史竣工但尚未归档的订单。**当前表没有 `par_month_id` 字段**；按 `act_date`（受理时间）或 `subs_stat_date`（竣工时间）查业务时间窗口时，不要把月表 `par_month_id` 当业务发生月；应先合并足够范围的归档月表与当前表，再用业务时间字段过滤，并按订单键去重。若为了合并输出需要归档月列，月表分支写 `cast(par_month_id as string) as archive_month`，当前表分支统一写 `cast(null as string) as archive_month`，或仅输出 `src_type='cur'`。

> ⏱️ **订单时间字段格式**：`act_date`、`subs_stat_date`、`open_date`、`start_date`、`end_date` 等订单时间字段常见值形如 `2009-12-03 11:06:59.0`。按自然日过滤时，优先写 `substr(cast(a.时间字段 as string),1,10) >= 'YYYY-MM-DD'` 这类日期截取条件，不要直接把带时分秒的字符串和日期常量混比。

## 本表常用条件速查（A 层附注）

| 场景 | 条件片段 |
|------|---------|
| 销售品发展量（含订购+互换） | `action_id IN (1292, 6200) AND COALESCE(subs_stat_reason,'-1') NOT IN ('1200','1300') AND date_format(subs_stat_date,'yyyyMM') IN ('YYYYMM',...)` |
| 历史业务时间销售品订单 | 月表 `par_month_id` 是归档月；当前表无 `par_month_id`；按 `act_date/subs_stat_date` 查历史业务时间时，先取足够范围的月表归档批次并 `UNION ALL` 当前表，再按业务时间字段过滤；订单时间字段按自然日过滤可用 `substr(cast(时间字段 as string),1,10)` |
| 是否竣工标记 | `CASE WHEN subs_stat='301200' THEN 1 ELSE 0 END AS is_jg`（默认标记列，不进 WHERE） |
| 排除撤单作废 | `COALESCE(subs_stat_reason,'-1') NOT IN ('1200','1300')` |
| 关联销售品维表 | `LEFT JOIN dws_crm_cfguse.dws_offer f ON a.prod_offer_id = f.offer_id AND f.city_id=200` |
| 关联 014 优惠资料在档实例 | `LEFT JOIN dwd_yz_rpt_comm_cm_msdisc_mon_final b ON a.msinfo_id = b.msobjgrp_id AND a.par_month_id = b.par_month_id`（041 的 `msinfo_id` 是 CRM 原始实例键；014 的 `msinfo_id` 已二次加工，勿默认用它关联） |
| 关联揽装局向 | `LEFT JOIN dwd_yz_dim_org lz_subst ON a.salestaff_subst_id = lz_subst.org_id AND lz_subst.levs=3`<br>`LEFT JOIN dwd_yz_dim_org lz_branch ON a.salestaff_branch_id = lz_branch.org_id AND lz_branch.levs=4` |

> 揽装信息口径：销售品办理的揽装人、揽装工号、揽装局向/营服/网点等归属信息在 041 订单过程表中；014 优惠资料表适合判断销售品在用/叠加结果，但不作为揽装信息主表。

参考 [RULES.md](../RULES.md)「口径与时间」「维表与 JOIN」「码值」相关规则；本文件中的常用条件仅作 041 表的快捷提示，规则权威以 `RULES.md` 为准。
