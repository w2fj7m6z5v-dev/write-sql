# CRM设备订单关系表

- **当前表 Hive 表名**：`dws_crm_order.dws_ord_prod_res_inst_rel`
- **归档表 Hive 表名**：`dws_crm_order.dws_ord_prod_res_inst_rel_his`
- **用途**：按订单标识回填入网时绑定的设备机身码；区别于 119 号码设备关系表（号码维度，当前状态）
- **典型入口**：用户需要取入网时设备信息，通过 069 的 `subs_id` 关联本表 `order_item_id`

## 字段说明

| 字段 | 字段类型 | 字段含义 | 说明 |
| ------ | ------ | ---------- | ------ |
| prod_inst_id | string | 产品实例ID / 服务标识 | 即 `serv_id`，`cast(prod_inst_id as decimal(22,0)) as serv_id` |
| order_item_id | string | 订单ID | 即 `subs_id`，**主键**，关联 069 的 `subs_id` |
| eqpt_sn | string | 设备机身码 | 需求口语"机身码" |
| status_cd | string | 状态 | `1100` = 有效；查询时必须加 `status_cd='1100'` |
| city_id | string | 地市 | 广州固定 `city_id=200` |

## 使用说明

- **当前表 + 归档表 UNION ALL**：归档表 `_his` 沉淀历史订单设备关系，当前表包含尚未归档的订单；查询时必须 UNION ALL 确保覆盖所有历史入网订单。
- **关联键**：`order_item_id` = 069 的 `subs_id`（入网订单标识），**不是** `prod_inst_id`。
- **与 119 号码设备关系表的区别**：
  - 119 按 `serv_id` 关联，取号码**当前**绑定的设备；
  - 本表按 `order_item_id` 关联，取**入网订单**绑定的设备；
  - 需要取"入网时的设备"必须走本表，119 只能取当前设备。
- **同一订单多设备**：一个入网订单可能绑定多台设备（如光猫+机顶盒），默认保留明细行；如只需光猫需加筛选条件。
- **状态过滤**：必须加 `status_cd='1100'`，排除无效记录。
- **城市过滤**：必须加 `city_id=200`。

## 常用链路

### 入网订单取设备机身码

```sql
-- 069 入网订单 → CRM 设备订单表取机身码
select
    o.acc_nbr,
    o.serv_id,
    o.subs_id,
    o.open_date,
    r.eqpt_sn
from tmp_install_order o
left join (
    select prod_inst_id, order_item_id, eqpt_sn
    from (
        select prod_inst_id, order_item_id, eqpt_sn
        from dws_crm_order.dws_ord_prod_res_inst_rel_his
        where city_id = 200 and status_cd = '1100'
        union all
        select prod_inst_id, order_item_id, eqpt_sn
        from dws_crm_order.dws_ord_prod_res_inst_rel
        where city_id = 200 and status_cd = '1100'
    ) t
    group by prod_inst_id, order_item_id, eqpt_sn
) r
  on o.subs_id = r.order_item_id;
```

## JOIN 与风险

- JOIN 键：`order_item_id`（订单维度），**不是** `prod_inst_id`（号码维度）。
- 同一 `order_item_id` 可能多条设备记录，默认保留明细；需一行时先确认聚合方式。
- 归档表 `_his` 需确认是否覆盖所有历史入网订单。
