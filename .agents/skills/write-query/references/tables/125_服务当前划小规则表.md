# 服务当前划小规则表

- **Hive 表名**: `dwd_yz_jyfx_serv_grid_final`
- **业务定位**: 补充服务当前划分规则、认领规则等规则结果；常与 069 当前责任田字段一起用于责任田归属审计。
- **不是**号码资料底座：号码、入网、状态、当前责任田 ID 等仍优先从 069 取。

## 已知字段

| 字段 | 字段类型 | 字段含义 | 说明 |
| ------ | ------ | --------- | ------ |
| serv_id | decimal(22,0) | 服务标识 | 与 069 `serv_id` 关联 |
| subst_rule | string | 当前划小局向 / 机构划分规则 | 可用于剔除“套餐规则”等业务确认的规则类型 |
| subst_date | string | 当前划小局向规则日期 | 规则生效/处理日期，按生产字段确认 |
| grid_rule | string | 当前责任田认领 / 划分规则 | 输出或审计字段 |
| grid_date | string | 当前责任田规则日期 | 规则生效/处理日期，按生产字段确认 |

## 使用规则

- 需要判断号码当前划分规则、是否由套餐规则等特定规则生成时，按 `069.serv_id = 本表.serv_id` 补本表。
- 责任田 ID、包区、营服、分局等当前归属字段，优先取 069 的 `grid_id/area_id/branch_id/subst_id`；本表主要补规则来源和规则日期。
- 规则类型过滤值需来自需求或稳定口径确认，不要把单个案例中的中文规则名无条件写死到所有查询。
