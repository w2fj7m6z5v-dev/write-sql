## 使用说明

- 查销售品是否在档、是否办理 / 是否叠加、销售品到期时间、套餐实例关系时，本表是存量事实表；本表放当月已竣工生效、已在用的销售品资料，适合回答”某账期是否已办某销售品”。
- 订单受理、竣工、归档、订购 / 发展动作不要改走本表，走 041 优惠订单表。041 是订单过程表，受理、竣工、归档之间有时间差；若用 041 查某竣工月，需按 `subs_stat='301200'` 和 `subs_stat_date` 控制业务月，并考虑当前表与历史月表覆盖差异。
- 需要历史账期快照、历史协议期、历史 `limit_date` 回溯时，用月表 `dwd_yz_rpt_comm_cm_msdisc_mon_final` 并锁 `par_month_id`。
- 协议期 / 合同年限核查可从本表取 `open_date`、`limit_date`、`prod_offer_id`，但具体协议销售品范围和年限系数应来自确认后的配置表或业务口径，不从案例号码硬编码。
- 实例键口径：本表 `msinfo_id` 已做二次加工，主从关系下可能用主实例覆盖从销售品实例；`msobjgrp_id` 保留 CRM 原始 `msinfo_id`。从 041 优惠订单表关联本表时，默认用 `041.msinfo_id = 014.msobjgrp_id`，月表还必须对齐 `par_month_id`。
- **实例粒度选择**：用户要”销售品实例”（一个客户可办理多个相同销售品）时，实例粒度用 `msobjgrp_id`（CRM 原始实例标识），不应按 `prod_offer_code` 去重聚合成一行。按实例分组时，同实例下号码用 `collect_set(acc_nbr)` 去重后 `concat_ws` 逗号拼接。
- **到期过滤默认**：查在档/在用销售品实例时，默认过滤未到期实例 `limit_date > current_date()`；仅用户明确要”含已到期”时移除该条件。

| load_date | timestamp | 加载时间 |

| prod_offer_code | string | 销售品编码 |
| prod_offer_name | string | 销售品名称 |
