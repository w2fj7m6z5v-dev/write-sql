# 直销客户表

## 用途

用于按产权客户编码补直销客户信息，例如直销客户编码、直销客户名称，以及直销客户表内记录的机构 ID。

适用范围是客户实体映射或客户信息维护，例如签订/维护直销客户、通过产权客户找直销客户。若需求是号码/服务明细，并希望输出号码对应的直销客户名或直销客户编码，优先使用 069 或当前事实主表自带客户字段。

## 字段

| 字段 | 类型 | 说明 |
|------|------|------|
| cust_nbr |  | 产权客户编码 |
| ccust_code |  | 直销客户编码 |
| ccust_name |  | 直销客户名称 |
| branch_org |  | 机构 ID；业务含义看来源字段和需求语境 |
| manage_org |  | 机构 ID；业务含义看来源字段和需求语境 |

## 常用关联

```sql
-- 产权客户编码 -> 直销客户信息
left join zone_gz_yz.dws_yz_tb_mo_custgrp_cust_final cc
  on seed.cust_nbr = cc.cust_nbr

-- 机构 ID -> 机构名称，按实际要解释的 ID 关联 018 机构维表
left join zone_gz_yz.dwd_yz_dim_org org1
  on cc.branch_org = org1.org_id
left join zone_gz_yz.dwd_yz_dim_org org2
  on cc.manage_org = org2.org_id
```

## 注意

- 本表是直销客户表，不是产权客户全量表；产权客户全量信息走 108。
- `branch_org`、`manage_org` 是机构 ID。机构维表只负责 `org_id` 到机构名称/层级字段的翻译，不能脱离来源字段语义直接断定为某类客户归属。
- 同一产权客户编码可能匹配多条直销客户记录；附件回填时保留原始行标识，并检查输入行数、命中行数、输出行数。
