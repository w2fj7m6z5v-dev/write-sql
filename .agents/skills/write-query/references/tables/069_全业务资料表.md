# 全业务资料表

- **Hive 表名（md 历史）**：`ads_yz_tb_comm_cm_all_final`
- **Hive 表名（日表 / 生产现网）**：`dwm_yz_tb_comm_cm_all_final` ← 近半年账期
- **Hive 表名（月表 / 生产现网）**：`dwm_yz_tb_comm_cm_all_mon_final` ← 历史账期（通常滞后一至两个月）
- **视图名**：`view_ads_yz_tb_comm_cm_all_final`

> ⚠️ **表名漂移与日/月表选择**：md 历史名前缀是 `ads_`。生产现网分日表与月表，**按整段查询账期判定**，不按「半年前 / 半年后」机械拆分 UNION。
>
> 1. 查询区间**完全落在日表近半年窗口**内 → **全程日表** `dwm_yz_tb_comm_cm_all_final`。
> 2. 否则 → 判断月表是否已覆盖**整段**所需账期（含 cohort 月、跟踪月等）；**能覆盖则全程月表** `dwm_yz_tb_comm_cm_all_mon_final`，重叠月份不必再 UNION 日表。
> 3. 仅当月表**缺**所需月份（常见：当月/最新月尚未入月表，如日表现网 202606、月表只到 202605）→ 月表已覆盖部分 + 日表补缺口，`UNION ALL`。
>
> 示例：cohort `202505` + 跟踪 `202506`~`202605`，跑数时点 `202606` → 月表全覆盖，**全程月表**；若跟踪要到 `202606` 而月表无 `202606` → 月表 `202506`~`202605` UNION 日表 `202606`。

## 本表常用条件速查（A 层附注）

> 全厂统一指标定义以 [指标索引](../METRIC_INDEX.md) 为准。下表为在本表上**常用的条件片段**备忘，供写 WHERE 时快速引用；与指标技术口径冲突时，以指标字典中的 **技术口径 (SQL)** 为准。

| 叫法/场景 | 条件片段 |
|----------|---------|
| 号码对应客户年龄 | `case when length(a.social_id) = 18 and a.social_id_type = '1' then cast(from_unixtime(unix_timestamp(),'yyyy') as integer)-cast(substr(a.social_id,7,4) as integer) when length(a.social_id) = 15 and a.social_id_type = '1' then cast(from_unixtime(unix_timestamp(),'yyyy') as integer)-cast('19' \|\| substr(a.social_id,7,2) as integer) else null end as age`；仅身份证证件类型计算，按当前年份减出生年份，不做生日月日精算；不输出身份证原文 |

## 时间口径提醒

- `par_month_id` 用于分区裁剪；入网、拆机、携入等业务发生时间必须同时用对应业务日期字段限定。
- 入网业务时间优先用 `open_date`；**主宽入网**（M-BASIC-BB-001/002）业务口语「按竣工时间」亦统一用 `open_date`，不用 `subs_stat_date`。
- **拆机默认（逻辑拆机）** 用 `hist_create_date`；物理拆机用 `wl_cancel_subs_stat_date`。
- **拆机标签按月快照**：`is_cancel_user=1` 仅出现在拆机当月 `par_month_id` 分区；反查拆机月时优先用 **月表** `dwm_yz_tb_comm_cm_all_mon_final`，`par_month_id` 即拆机月。
- **历史某月快照 / 拆机前 N 月属性**：与特性月表联查时，069 必须用月表，禁止日表 + 特性月表混配。
- 到达/出账类口径通常按账期状态判断，常用 `is_cancel_user=0` + `is_cz=1`。
| 当月逻辑拆机（默认口径） | `is_cancel_user=1`；拆机月=`par_month_id`；时间=`hist_create_date` |
| 当月物理拆机（特说明时） | `is_wl_cancel_user=1 and prod_type=40` |
