Architecture

Skill 的核心结构

一个文件夹，四个组成部分

my-skill / 目录结构
my-skill/ ├── SKILL.md ← 核心（必需）：元数据 + 指令 ├── references/ ← 知识库/规程（可选） ├── assets/ ← 模板/静态资源（可选） └── scripts/ ← 可执行脚本（可选）

关键点

只需一个 SKILL.md 文件，即可创建可用 Skill。极致轻量，极简封装。

案例

优化「Excel 转 Markdown」输出格式

Before

原始输出 (传统表格)

| 日期 | 驻点人员 | 县分 | |:---|:---|:---| | 25/7/1 | 黄娟娟 | 荔湾 |

问题：表格形式不利于知识库大模型 (RAG) 的语义检索和标签化管理。

After (期望)

Obsidian/RAG 风格

--- id: 007 tags: [#人员/黄娟娟, #县分/荔湾] --- # Q: 客编没业务怎么查？ ### 📋 基础元数据 - **驻点人员**: 黄娟娟

目标：每条独立成块，带 YAML 标签，完美适配知识库归档。

如何用自然语言完成改造？

第 2 轮
「把 Excel 转成适合 RAG 识别的 Markdown 格式，展示两条案例数据给我确认结构」
第 3 轮
「这个效果不是我想要的。我要每条独立成块，带 YAML 标签，适配知识库的检索。改完之后，请同样输出两条案例给我确认。」
第 4 轮
「把这次迭代的改进固化到这个 Skill」
Implementation

如何创建 Skill？

形成完整闭环的三种方式

1
推荐

对话式引导创建

场景：有需求但无从下手（如上述 Excel 归档）

优势：零代码起步，边聊边建，按需定制。
直接对 AI 提出需求，通过 4 轮对话，AI 自动生成配置和脚手架。

完整操作流程

  • 第 1 轮
    「利用 Find-Skills 查找 Excel 转 Markdown 的技能并安装」
    Find-Skills 搜索并安装
  • 第 2 轮
    「把 Excel 转成适合 RAG 识别的 Markdown 格式，展示两条案例数据给我确认结构」
    输出 2 条样例记录（表格格式）
  • 第 3 轮
    「这个效果不是我想要的。我要每条独立成块，带 YAML 标签，适配知识库的检索。改完之后，请同样输出两条案例给我确认。」
    调整为 YAML frontmatter + 章节划分，输出新样例确认
  • 第 4 轮
    「把这次迭代的改进固化到这个 Skill」
    生成可复用的 SKILL.md，后续自动按优化后的格式输出
2

基于 GitHub 开源项目

场景：借力成熟开源工具，快速封装视频下载能力

优势：5 分钟将成熟工具变为 AI 技能，复用开源生态。

操作流程

  • Step 1
    「帮我找一个 GitHub 上的视频下载开源项目」
    OpenClaw 返回 yt-dlp、you-get 等推荐
  • Step 2
    「基于这个链接创建 Skill：https://github.com/yt-dlp/yt-dlp」
    自动克隆项目、分析功能和使用方式、生成 triggers 并封装调用
  • Step 3
    直接使用：「下载这个视频 https://bilibili.com/xxx」
    Skill 自动调用 yt-dlp 完成下载
3

手工创建

场景：高级开发者，需要精细控制与深度定制

优势：完全可控。
手动创建目录及 SKILL.md，编写元数据、触发器、引用工具和脚本代码。
