# 销售品参数表

## 用途

用于补充销售品参数值，例如折扣、赠金、统付金额、优惠参数等。

**完整补表链路（069→014→107、JOIN 键、风险、自检）以 `FIELD_BACKFILL.md` §销售品参数值（107）为唯一权威**；本文件只保留字段与示例 SQL。

这张表只负责参数值补全，不负责判断销售品是否在档。在档 / 到期 → 014 优惠资料表 `ads_yz_rpt_comm_cm_msdisc_final`。

## 字段

| 字段 | 字段类型 | 说明 |
|------|------|------|
| serv_id | decimal(22,0) | 服务标识 |
| prod_offer_id | decimal(22,0) | 销售品标识 |
| param_code | varchar(64) | 参数编码 |
| param_value | varchar(300) | 参数值 |
| par_corp_id | decimal(10,0) | 地市分区 / 公司标识，广州固定 `200` |
| limit_date | timestamp | 到期时间 / 有效期字段（以生产表为准） |

## 常用条件

| 场景 | 条件 |
|------|------|
| 广州数据 | `par_corp_id = '200'` |
| 查询仍有效参数 | 按需求过滤 `limit_date >= 账期月底` |

## 常用关联

```sql
-- 1. 附件号码或需求号码先用 069 取 serv_id
-- 2. 用 014 查账期在档销售品和 prod_offer_id
-- 3. 再用本表补参数值
left join summary_ods_day_city.rpt_comm_cm_msparam p
  on msdisc.serv_id = p.serv_id
 and msdisc.prod_offer_id = p.prod_offer_id
 and p.param_code = ${param_code}
 and p.par_corp_id = '200'
```

## 注意

- `param_code` 必须来自用户、产品口径或已验证案例，不要猜参数编码。
- 同一 `serv_id + prod_offer_id` 可能存在多个参数；输出前确认是明细保留还是按参数编码打宽。
- 如果用户只问“是否办理 / 是否在档 / 到期时间”，不要用本表替代 014 优惠资料表。
