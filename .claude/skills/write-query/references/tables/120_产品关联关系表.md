# 产品关联关系表

- **Hive 表名 / 生产现网名**：`dws_crm_cust.dws_prod_inst_rel_a`
- **用途**：维护 A/Z 产品实例关联关系；既可按主端 / 群端服务实例找到 Z 端 / 子端服务，也可按成员 / 子端服务反查所属主端 / 群端服务。
- **典型入口**：用户提供群号、群端接入号、主端接入号、成员号码、子端接入号或对应 `serv_id`，要求找同组 A/B 端、Z 端、子端服务，或反查所属群端 / 主端服务，再查号码、状态、收入或拆机。

## 字段说明

| 字段 | 字段含义 | 说明 |
|------|----------|------|
| city_id | 地市编码 | 广州常用 `city_id='200'` |
| a_prod_inst_id | A 端 / 主端产品实例 ID | 通常由 069 `serv_id` 补得 |
| z_prod_inst_id | Z 端 / 子端产品实例 ID | 后续可回 069 补 `acc_nbr`、状态、客户、产品等 |

## 常用链路

### 群端 / 主端找 Z 端服务

```sql
select distinct
  rel.a_prod_inst_id,
  rel.z_prod_inst_id
from dws_crm_cust.dws_prod_inst_rel_a rel
where rel.city_id = '200'
  and rel.a_prod_inst_id in (select serv_id from tmp_seed_serv);
```

若附件只有接入号，先用 069 按账期补 `serv_id`，再关联本表。补出 `z_prod_inst_id` 后，常回 069：

```sql
left join dwm_yz_tb_comm_cm_all_final cm
  on rel.z_prod_inst_id = cm.serv_id
 and cm.par_month_id = ${month_id}
```

### Z 端 / 成员服务反查群端 / 主端服务

```sql
select distinct
  rel.a_prod_inst_id,
  rel.z_prod_inst_id
from dws_crm_cust.dws_prod_inst_rel_a rel
where rel.city_id = '200'
  and rel.z_prod_inst_id in (select serv_id from tmp_seed_serv);
```

若附件只有成员号码或子端接入号，先用 069 按账期补 `serv_id`，再用本表 `z_prod_inst_id` 反查 `a_prod_inst_id`。补出主端后，常回 069 取群端 / 主端接入号：

```sql
left join dwm_yz_tb_comm_cm_all_final cm_main
  on rel.a_prod_inst_id = cm_main.serv_id
 and cm_main.par_month_id = ${month_id}
```

## JOIN 与风险

- JOIN 键：主端找子端用 `serv_id = a_prod_inst_id`；子端反查主端用 `serv_id = z_prod_inst_id`；两端回资料表均用对应实例 ID 关联 069 `serv_id`。
- 常与 121 业务关联关系表合并使用，合并后必须按 `a_prod_inst_id,z_prod_inst_id` 去重。
- 一个主端可能多个 Z 端，一个子端也可能命中多个主端；默认保留明细。若用户要 A/B 两列、唯一群号或排序，需确认端别字段或业务口径，不能默认 `z_prod_inst_id` 小的是 A 端。
- 本表只承载产品实例关联关系，不直接提供接入号、客户、状态或收入字段。
