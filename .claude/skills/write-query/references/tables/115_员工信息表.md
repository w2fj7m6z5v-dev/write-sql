# 员工信息表

- **Hive 表名**：`dws_crm_cfguse.dws_staff`
- **业务含义**：员工信息表。指与中国电信存在雇佣关系的个人，包括内部员工和外部员工，例如支撑系统操作人员、合作伙伴员工、政企客户自助人员。
- **常用城市过滤**：广州数据常用 `city_id='200'`。

## 常用关联

```sql
-- 号码揽装人工号 -> 员工 CRM 账号
left join (
  select *
  from (
    select s.*,
           row_number() over (
             partition by s.staff_code
             order by s.status_date desc, s.update_ts desc
           ) as rn
    from dws_crm_cfguse.dws_staff s
    where s.city_id = '200'
      and s.staff_account like '11%'
  ) t
  where t.rn = 1
) staff
  on cm.sales_code = staff.staff_code
```

## 注意事项

- 补客户经理 11 开头 CRM 编码时，先从 069 全业务资料表按号码和账期取当前 `sales_code`，再用本表补 `staff_account/staff_name/staff_id`。
- 本表可能存在历史多版本记录，必须按 `status_date desc` 取最新；可用 `update_ts desc` 作为并列排序。
- `staff_account like '11%'` 是客户经理 CRM 编码场景的固定过滤。
- 本表补的是员工 CRM 工号，不替代 111 揽装人维表的有效性判断；需要判断揽装人是否有效时仍查 111。
