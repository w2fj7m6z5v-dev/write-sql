# 号码协销表

- **Hive 表名**: `zone_gz_yz.dwd_yz_cm_obj_xx_final`
- **历史月表**: `zone_gz_yz.dwd_yz_cm_obj_xx_mon_final`
- **视图名**: view_dwd_yz_cm_obj_xx_final
- **业务负责人**: 
- **业支负责人**: 杨洋
- **粒度**: 服务粒度协销人信息，以 `serv_id` 关联
- **使用规则**: 当前口径用 `_final`；历史账期用 `_mon_final` 并按 `par_month_id` 对齐

---

## 字段说明

| 字段 | 字段类型 | 字段含义 |
|------|---------|---------|
| 字段 |  | 字段名称 |
| serv_id |  | 服务标识 |
| xx_salestaff_id1 |  | 第二发展人ID |
| xx_salestaff_code1 |  | 第二发展人编码 |
| xx_salestaff_name1 |  | 第二发展人名称 |
| xx_salestaff_id2 |  | 第三发展人ID |
| xx_salestaff_code2 |  | 第三发展人编码 |
| xx_salestaff_name2 |  | 第三发展人名称 |

## 字段口径

- `xx_salestaff_id1/code1/name1`：第一协销人，也常被称为第二发展人。
- `xx_salestaff_id2/code2/name2`：第二协销人，也常被称为第三发展人。
- 只有 `serv_id` 或号码清单补协销人时，优先用本表；若主表已有订单键并明确要订单协销，使用 043 订单协销表。

## 常用链路

### 服务清单补协销人

```sql
select
  a.*,
  b.xx_salestaff_id1,
  b.xx_salestaff_code1,
  b.xx_salestaff_name1,
  b.xx_salestaff_id2,
  b.xx_salestaff_code2,
  b.xx_salestaff_name2
from tmp_seed a
left join zone_gz_yz.dwd_yz_cm_obj_xx_final b
  on a.serv_id = b.serv_id;
```

历史账期使用 `zone_gz_yz.dwd_yz_cm_obj_xx_mon_final`，并补充 `a.par_month_id = b.par_month_id` 或用指定账期过滤。

## JOIN 与风险

- JOIN 键：`serv_id`。
- 历史账期必须使用月表并对齐 `par_month_id`。
- 服务粒度协销不要反向改订单主表；订单粒度协销使用 043。
