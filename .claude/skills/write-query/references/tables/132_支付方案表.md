# 支付方案表

- **Hive 表名 / 生产现网名**：`dws_crm_cust.dws_payment_plan`
- **用途**：描述用户费用的支付方案；按缴费账户 `acct_id` 找到外部支付账户 `pay_acct_id`。
- **上游入口**：131 产品实例账户关系表产出的 `acct_id`，或用户清单已有 `acct_id`。
- **默认过滤**：广州固定 `city_id=200`。

## 字段说明

| 字段 | 字段类型 | 字段含义 | 说明 |
| ------ | ------ | ---------- | ------ |
| payment_plan_id | string | 支付方案标识 | 支付方案主键 |
| acct_id | string | 支付 / 缴费账户标识 | 上游账户键 |
| pay_acct_type | string | 支付账户所属类型 | 码值含义需按生产字典确认 |
| pay_acct_id | string | 支付账户标识 | 关联 133 外部账户表 `ext_acct_id` |
| payment_method | string | 付款方式 | 码值含义需按生产字典确认 |
| priority | string | 优先级 | 多支付方案取一条时可作为候选排序字段，需用户确认 |
| status_cd | string | 记录支付方案的状态 | 状态码值需确认 |
| eff_date | string | 生效时间 | 历史/有效性判断候选字段 |
| exp_date | string | 失效时间 | 历史/有效性判断候选字段 |
| city_id | string | 地市 | 广州固定 `city_id=200` |

## 常用链路

```sql
select
  acct.acct_id,
  plan.pay_acct_id,
  plan.payment_method,
  plan.priority
from tmp_acct acct
left join dws_crm_cust.dws_payment_plan plan
  on acct.acct_id = plan.acct_id
 and plan.city_id = 200;
```

## JOIN 与风险

- `acct_id` 下可能存在多条支付方案；默认保留明细，不擅自按 `priority/status_cd/eff_date/exp_date` 去重。
- 用户要求一账户一条或一服务一条时，必须确认有效状态、优先级和时间口径。
- 本表不直接提供银行账户名称；账户户名在 133 外部账户表 `pay_acct_name`。
