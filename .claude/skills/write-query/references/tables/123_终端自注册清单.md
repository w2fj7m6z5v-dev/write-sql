# 终端自注册清单

- **Hive 表名**：`summary_ods_day_szx.rpt_terminal_type_new`
- **用途**：按注册号码回填终端自注册信息，例如集团自注册机型、终端制式、手机厂商、标准化机型、注册时间、IMSI/IMEI 和串号。
- **典型入口**：附件号码、069 全业务资料表或其它号码明细中的 `acc_nbr`。
- **不要混用**：设备资源字段（设备名称、设备类型、购买方式、机身号、数量）走 119 设备资源关系表；终端装维成本走 090；产品规格 / 附属产品特性走 105/106。

## 字段说明

| 字段 | 字段类型 | 字段含义 |
|------|----------|----------|
| sum_day | varchar(15) | 处理日期 |
| corp_id | varchar(11) | 分公司区号 |

| source_type | varchar(11) | 数据来源 |
| phone_strcode | varchar(64) | 原始终端注册串号 |
| register_time | timestamp | 号码最新注册时间 |
| intelligence | decimal(6,0) | 是否智能机 |
| std_phone_strcode | varchar(16) | 标准化串号 |
| sys_time | timestamp | 数据加载时间 |
| imsi | varchar(30) | IMSI 码 |
| imei1 | varchar(30) | IMEI1 码 |
| imei2 | varchar(30) | IMEI2 码 |

## 常用 SQL

### 号码清单补最新终端自注册信息

```sql
select
  a.acc_nbr,
  t.terminal_type,
  t.brand_type,
  t.factory,
  t.brand,
  t.register_time,
  t.imsi,
  t.imei1,
  t.imei2
from seed_acc_nbr a
left join (
  select
    acc_nbr,
    terminal_type,
    brand_type,
    factory,
    brand,
    register_time,
    imsi,
    imei1,
    imei2,
    row_number() over(partition by acc_nbr order by register_time desc, sys_time desc) as rn
  from summary_ods_day_szx.rpt_terminal_type_new
) t
  on a.acc_nbr = t.acc_nbr
 and t.rn = 1
;
```

## 风险与审计

- 本表按 `acc_nbr` 关联，和 119 设备资源关系表按 `serv_id` 关联不同。
- 同一号码可能多次注册；明细需求可保留多行，一号一行默认按 `register_time desc, sys_time desc` 取最新。
- 如果需求要求“入网附近注册终端”，可在补最新记录后增加与 069 `open_date` 的时间差条件；不要把固定天数写死，除非需求或口径明确。
- 补表后核对附件行数、命中号码数，以及按 `acc_nbr` 是否放大。
