# 全业务号码订单表

- **Hive 表名（当前表）**: `zone_gz_yz.dwm_yz_rpt_comm_ba_subs_final`
- **月表 / 历史归档表名**: `zone_gz_yz.dwm_yz_rpt_comm_ba_subs_mon_final`
- **视图名**: `view_dwm_yz_rpt_comm_ba_subs_final`
- **业务负责人**: 
- **业支负责人**: 杨洋

> ⚠️ **当前表 vs 月表**：订单生命周期为“受理 → 竣工 → 归档”。月表 `dwm_yz_rpt_comm_ba_subs_mon_final` 只沉淀已归档订单，`par_month_id` 表示**归档月**，不是受理月，也不严格等于竣工月；例如 `par_month_id=202603` 表示 2026 年 3 月归档的号码订单，其中既可能有 3 月受理/竣工/归档订单，也可能有历史受理或历史竣工、3 月才归档的订单。
>
> 当前表 `dwm_yz_rpt_comm_ba_subs_final` 是当前订单池，包含当前月归档订单，以及历史受理/历史竣工但尚未归档的订单。**当前表没有 `par_month_id` 字段**；按 `act_date`（受理时间）或 `subs_stat_date`（竣工时间）查业务时间窗口时，不要把月表 `par_month_id` 当业务发生月；应先合并足够范围的归档月表与当前表，再用业务时间字段过滤，并按订单键去重。若为了合并输出需要归档月列，月表分支写 `cast(par_month_id as string) as archive_month`，当前表分支统一写 `cast(null as string) as archive_month`，或仅输出 `src_type='cur'`。

> ⏱️ **订单时间字段格式**：`act_date`、`subs_stat_date`、`serv_create_date` 等订单时间字段常见值形如 `2009-12-03 11:06:59.0`。按自然日过滤时，优先写 `substr(cast(a.时间字段 as string),1,10) >= 'YYYY-MM-DD'` 这类日期截取条件，不要直接把带时分秒的字符串和日期常量混比。
