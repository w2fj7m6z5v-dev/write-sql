# 敏感客户黑名单表

## 用途

用于查询客户级或证件级敏感黑名单。支持按 `cust_id` 判断客户是否在黑名单中，也支持按证件号反查客户黑名单状态。

## 字段

| 字段 | 字段类型 | 说明 |
|------|------|------|
| obj_id | string | 黑名单对象 ID；客户级时为 `cust_id`，证件级时为证件号 |
| obj_type | string | 对象类型：`1100`=客户级，`1500`=证件级 |
| special_type | string | 特殊名单类型：`1200`=敏感客户黑名单 |
| sub_special_type | string | 子类型（细分黑名单类别） |
| status_cd | string | 状态：`1000`=有效 |
| create_date | string | 创建时间 |
| create_staff | string | 创建人工号 |
| city_id | string | 城市 ID（广州=200） |

## 常用条件

```sql
-- 客户级黑名单（obj_id 即 cust_id）
select obj_id as cust_id, sub_special_type, create_date
from dws_crm_party.dws_special_list_black
where special_type = '1200'
  and obj_type = '1100'
  and status_cd = '1000'
  and city_id = 200
  and obj_id > 0 and obj_id is not null;

-- 证件级黑名单（cert_nbr 即证件号）
select cert_nbr, sub_special_type, create_date
from dws_crm_party.dws_special_list_black
where special_type = '1200'
  and obj_type = '1500'
  and status_cd = '1000'
  and city_id = 200
  and cert_nbr is not null;
```

## 常用关联

```sql
-- 客户级黑名单直接关联 cust_id
left join (
  select obj_id as cust_id, sub_special_type
  , row_number() over(partition by obj_id order by create_date desc) as rn
  from dws_crm_party.dws_special_list_black
  where special_type = '1200' and obj_type = '1100' and status_cd = '1000' and city_id = 200
) blk on 主表.cust_id = blk.cust_id and blk.rn = 1
```

## 注意

- 客户级和证件级是两条独立链路，最终需合并去重（`union all` 后按 `cust_id` 取最新）。
- 证件级黑名单需经过 136 证件本地表 + 108 产权客户表才能得到 `cust_id`。
- 同一 `obj_id` 可能有多条历史记录，默认按 `create_date desc` 取最新。
- 不要与 122 名单制管控清单混用；122 是直销客户名单制行商类型，本表是敏感客户黑名单。
