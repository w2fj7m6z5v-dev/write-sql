# 特性资料表（产品规格属性）

- **Hive 表名（日表 / 在网）**：`summary_ods_day_city.tb_pre_cm_attr_all` — 当前在网号码**产品规格**特性
- **Hive 表名（月表 / 历史）**：`iodata_ods_month_city.tb_pre_cm_attr_all_mon` — 历史月份产品规格特性快照

> ⚠️ **勿与附属产品属性混用**：附属产品特性走 [106 附属产品资料表](106_附属产品资料表.md)（`rpt_comm_cm_subserv` / `_mon`）。用户说「属性」且上下文有 `sub_prod_id` / 「附属产品」→ 106；否则默认本表（105）。
>
> ⚠️ **日表 vs 月表**：日表只反映**当前在网**号码；已拆机或需取**历史某月**特性快照时，必须用月表。月表当前账期通常不含当月（如当前 202605，月表存 202604 及更早）。

## 字段说明

| 字段 | 字段含义 | 说明 |
|------|---------|------|
| serv_id | 服务标识 | 关联 069、种子表等 |
| attr_id | 特性 ID（产品规格属性编码） | 如 `500040209` |
| attr_value1 | 特性值（码值） | 输出字段；中文名补字典 |
| create_date | 订购时间 | 特性订购/变更时间 |
| par_corp_id | 城市/公司分区 | 广州固定 `200` |
| par_month_id | 统计月份（月表） | 属性快照账期 |

## 本表常用条件速查

| 场景 | 条件 |
|------|------|
| 广州特性 | `par_corp_id='200'` |
| 历史某月特性快照 | 月表 `par_month_id=${attr_month_id}` |
| 指定产品规格属性 | `attr_id IN ('500040209', ...)` |
| IMSI | `attr_id='200000103'`，输出 `attr_value1` |
| 欠费停机属性 / 欠停时间 | `char_class='04' AND attr_id=98`，输出 `create_date`；默认按 `create_date asc` 取首次欠费停机时间 |
| 拆机前一月特性 | 先 069 月表定 `cancel_month_id`，再 `attr_month_id = cancel_month_id - 1`，关联月表 |

## 特性值中文名

补 `dws_crm_cfguse.dws_attr_value`（`tables/015_字典表视图.md`）：

```sql
a.attr_id = b.attr_id
AND a.attr_value1 = b.attr_inner_value
AND b.city_id = '200'
```

> 产品规格属性用 **`attr_inner_value`** 关联，不是 069 `state` 用的 `attr_value`。

## 注意事项

- 同一 `serv_id + attr_id` 在月表给定账期内通常唯一；若业务确认有多行，按 `create_date` 取最新。
- 欠费停机属性时间：用 `char_class='04' AND attr_id=98`；按服务回填时默认 `row_number() over(partition by serv_id order by create_date asc)=1` 取首次记录。
- 不要用日表取已拆机号码的历史特性。
- 根据号码清单导 IMSI 时，先用 069 按 `acc_nbr + par_month_id` 定位 `serv_id`，再取本表 `attr_id='200000103'` 的 `attr_value1`。
- 详见 `verified-cases/VC-20260522-001_种子serv_id拆机前月产品规格属性宽表.md`。
