# 证件本地表

## 用途

用于按证件号（身份证号等）反查 `party_id`，再通过 108 产权客户表获取 `cust_id`。常见场景：证件级黑名单反查客户、身份证号匹配客户名下号码。

## 字段

| 字段 | 字段类型 | 说明 |
|------|------|------|
| party_id | string | 参与方 ID，关联 `dws_customer.party_id` |
| cert_type | string | 证件类型：`1`=身份证 |
| cert_num | string | 证件号码 |

## 常用条件

```sql
-- 按身份证号查 party_id
select party_id, cert_num
from dws_crm_cust.dws_party_cert_local
where cert_type = '1'  -- 身份证
group by party_id, cert_num;
```

## 常用关联

```sql
-- 证件号 → party_id → cust_id（证件级黑名单反查客户）
-- 步骤1：证件号 → party_id
left join (select party_id, cert_num from dws_crm_cust.dws_party_cert_local where cert_type='1' group by party_id, cert_num) cert
  on 黑名单.cert_nbr = cert.cert_num

-- 步骤2：party_id → cust_id
left join (select cust_id, party_id from dws_crm_cust.dws_customer where city_id=200) cust
  on cert.party_id = cust.party_id
```

## 注意

- 本表只存证件号与 `party_id` 的映射，不含客户名称或编码；客户信息需通过 108 产权客户表补全。
- 同一证件号可能对应多个 `party_id`（历史变更），默认保留明细。
- 不要用本表替代 069 的 `social_id` 字段；069 已加工好证件号→服务对象的映射，常规查询直接用 069。
