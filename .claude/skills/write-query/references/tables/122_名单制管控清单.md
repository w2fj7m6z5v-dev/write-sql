# 名单制管控清单

## 用途

用于在需求明确要求“名单制用管控清单，不用主表 `is_mdz`”时，按直销客户编码补名单制管控信息。

若主表已经有直销客户编码字段，例如双线清单的 `cust_code`，可直接关联本表。若只有产权客户编码 `cust_nbr`，先补 109 直销客户表取得 `ccust_code`，再关联本表。

## 字段

| 字段 | 字段类型 | 说明 |
|------|------|------|
| ccust_code | string | 直销客户编码 |
| hk_flag | string | 名单制行商类型 / 名单制标签 |
| create_date | string | 名单制创建时间 |

## 常用关联

```sql
-- 主表已有直销客户编码
left join ads_yz_mo_ccust_mdz_final mdz
  on main.cust_code = mdz.ccust_code

-- 只有产权客户编码时，先 109 补直销客户编码
left join zone_gz_yz.dws_yz_tb_mo_custgrp_cust_final cc
  on main.cust_nbr = cc.cust_nbr
left join ads_yz_mo_ccust_mdz_final mdz
  on cc.ccust_code = mdz.ccust_code
```

## 注意

- 本表用于“管控名单制清单”口径，不等同于 069、033 或收入表中的 `is_mdz` 标签。
- 只有用户明确要求用名单制管控清单、或当前需求上下文已指定该口径时，才优先使用本表。
- 直销客户编码与产权客户编码不能混用；来源只有产权客户编码时先走 109。
- 关联后检查输入行数、命中行数和输出行数，避免同一客户多记录导致明细放大。
