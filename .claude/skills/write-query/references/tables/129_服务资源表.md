# 服务资源表

- **Hive 表名 / 生产现网名**：`dws_crm_cust.dws_cust_serv_res`
- **历史表**：`dws_crm_cust.dws_cust_serv_res_his`
- **用途**：按服务实例回填 DP、ONU、OBD、主干、LAN、交接箱等服务资源编码。
- **典型入口**：用户提供 `serv_id` / `prod_inst_id` 清单，或附件只有号码时先用 069 补 `serv_id`。
- **默认过滤**：广州固定 `city_id='200'`。

## 字段说明

| 字段 | 字段含义 | 说明 |
|------|----------|------|
| prod_inst_id | 产品实例标识 | 生成 SQL 时通常 `cast(prod_inst_id as decimal(22,0)) as serv_id` |
| dp_id | CRM 系统内分线盒标识 | 分线盒内部 ID |
| dp_code | 分线盒编码 / DP 编码 | 需求口语“DP 编码” |
| box_id | CRM 系统内部交接箱标识 | 交接箱内部 ID |
| box_code | 交接箱编码 | 需求口语“交接箱编码” |
| trunk_id | CRM 系统内部主干标识 | 主干内部 ID |
| trunk_code | 主干编码 | 需求口语“主干编码” |
| onu_code | ONU 编码 | 需求口语“ONU 编码” |
| obd_code | OBD 编码 | 需求口语“OBD 编码” |
| lan_code | LAN 交换机编号 | 需求口语“LAN 交换机编号” |
| org_id | 机构标识 | 资源记录归属机构 |
| post_id | 工位 | 资源记录工位 |
| status_cd | 记录状态 | 状态码值；含义需按字典或生产说明确认 |
| status_date | 状态变更时间 | 记录状态变更时间 |
| create_date | 记录创建时间 | 创建时间 |
| update_date | 记录修改时间 | 修改时间 |
| city_id | 地市 | 广州固定 `city_id='200'` |

## 常用链路

### 服务清单补 DP/ONU/OBD 编码

```sql
select
  seed.serv_id,
  res.dp_code,
  res.onu_code,
  res.obd_code,
  res.box_code,
  res.trunk_code,
  res.lan_code
from tmp_seed seed
left join dws_crm_cust.dws_cust_serv_res res
  on cast(seed.serv_id as decimal(22,0)) = cast(res.prod_inst_id as decimal(22,0))
 and res.city_id = '200';
```

附件只有接入号时，先用 069 按账期补 `serv_id`，再关联本表。

历史回溯时使用 `dws_crm_cust.dws_cust_serv_res_his`。若需要按历史时点取数，必须按实际历史表时间字段确认生效/失效或状态变更口径，不要只替换表名。

## JOIN 与风险

- JOIN 键：`serv_id = prod_inst_id`，建议两侧统一转 `decimal(22,0)` 或同一字符类型后关联。
- 当前表默认限制 `city_id='200'`。
- 同一 `prod_inst_id` 如存在多条资源记录，默认保留明细；若要一服务一行，需确认按 `status_date/update_date` 取最新还是按状态过滤。
- 本表只承载服务资源编码，不用于判断设备名称、设备购买方式、机身号或设备数量；这些字段走 119 设备资源关系表。
