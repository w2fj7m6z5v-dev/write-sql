# 移机订单表

- **Hive 表名**: `dwd_yz_rpt_comm_ba_subs_move_final`
- **生产表名**: `dwd_yz_rpt_comm_ba_subs_move_final`
- **业务含义**: 移机订单明细表，用于查询移机成功订单、竣工时间，以及移机前后落地局向、营服、网格单元。
- **粒度**: 移机订单 / 接入号 / 竣工时间粒度；同一接入号在一个时间窗内可能存在多条移机订单。
- **分区字段**: `par_month_id`

---

## 常用场景

- 按投诉号码清单匹配投诉归档日前后 N 天内是否存在移机成功订单。
- 按宽带接入号查询移机订单编码、竣工时间。
- 输出移机前后所属落地分局、落地营服、网格单元。

---

## 核心字段

| 字段 | 字段含义 |
|------|----------|
| par_month_id | 分区月份 |
| serv_id | 服务标识 |
| acc_nbr | 移机订单对应接入号 |
| subs_code | 移机订单编码 |
| subs_stat_date | 移机订单竣工时间 / 状态完成时间 |
| std_subst_name_last | 移机前所属落地局向 |
| std_branch_name_last | 移机前所属落地营服 |
| cell_name_last | 移机前所属网格单元 |
| std_subst_name | 移机后所属落地局向 |
| std_branch_name | 移机后所属落地营服 |
| cell_name | 移机后所属网格单元 |

---

## 使用建议

- 分区范围要覆盖业务时间窗。例如投诉归档月份为 `202505-202601`，前后 30 天匹配时，移机表分区至少取 `202504-202602`。
- 投诉号码为宽带号时，可直接用附件号码关联本表 `acc_nbr`。
- 投诉号码为移动号时，移动号码本身不产生移机订单，应先用 069 月表按 `rh_tc_id` 找同融合套内宽带号码，再用宽带号关联本表。
- 多条订单命中时，默认取最接近投诉归档日期的一条：按 `abs(datediff(投诉归档日期, subs_stat_date))` 升序排序。

---

## 注意事项

- 本表承接移机订单事实，不用于普通入网、到达、在网规模统计。
- 时间窗匹配必须同时写分区裁剪和业务时间条件；分区只控制读数范围，不能替代 `subs_stat_date` 与投诉归档日期的日期比较。
- 附件驱动时保留投诉工单号、附件序号或原始号码，输出前核对输入行数、宽带转换命中行数、订单匹配行数。
