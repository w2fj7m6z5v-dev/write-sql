# 市场化合同有效揽装人

## 适用

- 用户按市场化承包合同、合同编码、合同网点查询有效揽装人或实际工号数量。
- 用户诊断网点为什么没有号码或收入，需要区分无效网点、有效网点无揽装人、揽装人无效。

## 不适用

- 普通号码当前揽装人：优先看 069 或当前事实主表字段。
- 客户经理 CRM 编码：用 `SC-001_客户经理CRM编码.md`。
- 普通收入明细：按收入路由选择 047/048/117 等收入事实表。

## 主表与补表

| 角色 | 表 | 用途 |
|---|---|---|
| 合同网点事实 | 110 结算账单表 `dws_tpss_jszx.dws_settle_bill` | 合同、结算账期、网点事实 |
| 有效组合 | 113 揽装所属月表 `dwd_yz_sales_man_outlers_mon_final` | 有效网点 + 有效揽装人关系 |
| 网点有效性 | 112 网点月表 `dwd_yz_sale_outlers_mon_final` | 判断网点是否无效 |
| 揽装人有效性 | 111 揽装人月表 `dwd_yz_sales_man_mon_final` | 判断揽装人是否无效 |

## 有效揽装人查询规则

- 合同、结算账期、网点事实来自 110，不是 069、订单表、积分表或收入明细表。
- 110 常用字段：`contractno`、`contract_name`、`billing_cycle_id`、`channel_id`、`channel_code`、`channel_name`。
- 广州常用 `shard='200'`。
- 历史账期补有效揽装人时，优先补 113 月表。
- JOIN：`settle.channel_id = smo.channel_id` AND `substr(settle.billing_cycle_id,1,6)=smo.par_month_id`。
- 明细输出可保留多名有效揽装人；若用户问实际工号数量，优先 `count(distinct smo.staff_id)`。
- `sales_code` 不唯一，不能作为揽装人唯一 JOIN 或去重键。

## 无号码收入网点诊断

1. 先用 112 网点月表按 `par_month_id + channel_nbr/channel_id` 判断网点有效性；`status_cd='S0X'` 为无效网点。
2. 有效网点若在 113 揽装所属月表同账期无记录，判定为有效网点但无有效揽装人关系。
3. 需要细分揽装人无效时，用 111 揽装人月表按 `staff_id` 回查；`status_cd='S0X'` 为无效揽装人。
4. 业务解释：无效网点、有效网点但无揽装人、有效网点但揽装人无效，均不会发展号码和收入，因为网点号码和收入归属通过有效揽装人打标。

## 风险审计

- 不要只因 113 无记录就断言网点不存在。
- 历史账期必须使用 111/112/113 的 `_mon_final` 月表并对齐 `par_month_id`。
- 实际工号数量按 `staff_id` 去重，不按 `sales_code`。
