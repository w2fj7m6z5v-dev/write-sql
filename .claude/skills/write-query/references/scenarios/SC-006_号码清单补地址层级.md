# 号码清单补地址层级

## 适用

- 用户给号码、宽带号码或接入号清单，要求回填标准装机地址所属 7 级地址 ID/名称。
- 用户要求按标准装机地址上卷到 5 级地址 ID/名称。

## 不适用

- 只需要主表已有装机地址字段：先看主表字段。
- 非标准地址层级或用户自定义地址口径：先确认来源。

## 主表与补表

| 角色 | 表 | 用途 |
|---|---|---|
| 驱动表 | 用户附件号码清单 | 保留原始序号和号码 |
| 主表 | 069 全业务资料表 | 按号码和账期取 `serv_id/serv_addr_id` |
| 补表 | 079 地址维表 `zone_gz_yz.dwd_yz_addr_final` | 装机地址层级上卷 |

## 7 级地址路径

```text
069.serv_addr_id
  -> 079.id
  -> 079.addr_id_7
  -> 079.id (grade=7)
```

关键写法：

- `069.serv_addr_id = cast(addr.id as string)`。
- `cast(addr.addr_id_7 as string) = cast(addr7.id as string)`。
- 输出 7 级地址名称时第二次关联必须加 `addr7.grade=7`。

## 5 级地址路径

```text
serv_addr_id
  -> addr10.id
  -> addr10.addr_id_6
  -> addr6.id
  -> addr6.parentid
  -> addr5.id
```

关键写法：

- 先用主表 `serv_addr_id` 关联 079 的 `id`，取 `addr_id_6`。
- 再用 `addr_id_6` 关联 079 的 `id`，取该 6 级地址行的 `parentid` 作为 5 级地址 ID。
- 最后用 `parentid` 关联 079 的 `id`，取 `addr` 作为 5 级地址名称；建议加 `addr5.grade=5`。

## 硬规则

- 地址 ID 关联统一转字符。
- 禁止默认写 `cast(069.serv_addr_id as decimal(24,0)) = addr.id`，长地址 ID 或异常字符可能导致转换失败或漏数。
- 附件驱动需核对输入行数、069 命中行数、地址命中行数、输出行数。
