# VC-20260713-001：营服净增分析 — 流入+保持-流失（并集打标宽表法）

## 适用问题

营服/分局级净增分析，需拆解为流入、保持、流失三部分，并区分细分原因：
- 流失：拆机（物理/逻辑）、计转非、转去同分公司其他营服、转去其他分公司、号码不存在
- 流入：新入网、从同分公司其他营服转入、从其他分公司转入、新到达
- 保持：仍在该营服到达

## 不适用

- 全市级净增（跨机构转移可忽略，直接用到达差值）
- 纯到达量对比（不做流失归因）
- 纯拆机清单（不做流入分析）

## 核心思路

### 1. 净增公式

```
净增 = 流入 + 保持 - 流失
上月到达 = 保持 + 流入
拍照月到达 = 保持 + 流失
```

### 2. 并集打标法（而非分别做流失+流入两张表）

```
步骤1：圈拍照月该营服到达号码（集合 A）
步骤2：圈上月该营服到达号码（集合 B）
步骤3：并集 = A ∪ B（按 serv_id 去重，snap_flag/cur_flag 标记归属）
步骤4：月份事件日志（逐月扫描 is_new_user=1 / is_cancel_user=1）
步骤5：打标宽表 = 并集 LEFT JOIN 拍照月全量 + 上月全量 + 事件日志
        → 一条 CASE WHEN 判定 flow_type + tag_reason
步骤6：汇总（GROUP BY flow_type, tag_reason）
```

**优势**：打标宽表 `tmp_th_bb_tagged_wide` 是后续清单查询的基础，直接 `WHERE flow_type='流失' AND tag_reason LIKE '%拆机%'` 即可。

### 3. 关键修正：069 表存放逻辑

> 069 每张表只存当月状态，不能简单用"号码存在/不存在"判断入网或拆机。

正确做法：
- **入网判断**：逐月扫描 `is_new_user=1`，建立入网事件日志
- **拆机判断**：逐月扫描 `is_cancel_user=1`，建立拆机事件日志
- **跨机构转移**：号码在上月全量中存在且在到达（`is_cancel_user=0 and is_cz=1`），但 `branch_name` 与拍照月不同 → 转去其他营服/分公司
- **号码不存在**：上月全量中找不到 + 事件日志无记录 → 可能已销户或月表未覆盖

### 4. 营服级 vs 全市级

营服级净增必须考虑跨机构转移：
- 号码从该营服转到其他营服 → 对该营服是流失，对全市不是
- 号码从其他营服转入该营服 → 对该营服是流入，对全市不是

全市级净增中跨机构转移互相抵消，可直接用到达差值。

### 5. 拍照月

拍照月 = 上年12月（如当前2026年取 `202512`），不是去年同期。
详见 ROUTING.md 术语映射「拍照月」。

## SQL 编排模板

### 步骤1：拍照月该营服到达号码（月表）

```sql
drop table if exists tmp_{scope}_snapshot_serv purge;
create table tmp_{scope}_snapshot_serv ... as
select a.serv_id, a.acc_nbr, a.subst_name, a.branch_name
from dwm_yz_tb_comm_cm_all_mon_final a
where a.par_month_id = ${snapshot_month}    -- 拍照月，如 202512
  and a.subst_name = '${subst_name}'        -- 分公司
  and a.branch_name = '${branch_name}'      -- 营服
  and a.prod_type = 40
  and a.kd_desc = '普通宽带'
  and a.is_cancel_user = 0
  and a.is_cz = 1;
```

### 步骤2：上月该营服到达号码（日表）

```sql
drop table if exists tmp_{scope}_current_serv purge;
create table tmp_{scope}_current_serv ... as
select a.serv_id, a.acc_nbr, a.subst_name, a.branch_name
from dwm_yz_tb_comm_cm_all_final a
where a.par_month_id = ${current_month}     -- 上月，如 202606
  and a.subst_name = '${subst_name}'
  and a.branch_name = '${branch_name}'
  and a.prod_type = 40
  and a.kd_desc = '普通宽带'
  and a.is_cancel_user = 0
  and a.is_cz = 1;
```

### 步骤3：并集（按 serv_id 去重）

```sql
drop table if exists tmp_{scope}_union_serv purge;
create table tmp_{scope}_union_serv ... as
select serv_id, max(acc_nbr) as acc_nbr,
       max(snap_flag) as snap_flag, max(cur_flag) as cur_flag
from (
    select serv_id, acc_nbr, 1 as snap_flag, 0 as cur_flag
    from tmp_{scope}_snapshot_serv
    union all
    select serv_id, acc_nbr, 0 as snap_flag, 1 as cur_flag
    from tmp_{scope}_current_serv
) t
group by serv_id;
```

### 步骤4：月份事件日志（逐月入网+拆机）

覆盖拍照月到上月之间的所有月份。月表取历史月，日表取最近月。

```sql
drop table if exists tmp_{scope}_monthly_event_log purge;
create table tmp_{scope}_monthly_event_log ... as
select par_month_id, serv_id, event_type, is_wl_cancel_user
from (
    -- 入网事件（月表覆盖段）
    select par_month_id, serv_id, 'new_install' as event_type, 0 as is_wl_cancel_user
    from dwm_yz_tb_comm_cm_all_mon_final
    where par_month_id between ${event_start_month} and ${event_end_month_mon}
      and is_new_user = 1
      and prod_type = 40 and kd_desc = '普通宽带'
    union all
    -- 入网事件（日表补缺口）
    select par_month_id, serv_id, 'new_install' as event_type, 0 as is_wl_cancel_user
    from dwm_yz_tb_comm_cm_all_final
    where par_month_id = ${event_current_month}
      and is_new_user = 1
      and prod_type = 40 and kd_desc = '普通宽带'
    union all
    -- 拆机事件（月表覆盖段）
    select par_month_id, serv_id, 'disconnect' as event_type, is_wl_cancel_user
    from dwm_yz_tb_comm_cm_all_mon_final
    where par_month_id between ${event_start_month} and ${event_end_month_mon}
      and is_cancel_user = 1
      and prod_type = 40 and kd_desc = '普通宽带'
    union all
    -- 拆机事件（日表补缺口）
    select par_month_id, serv_id, 'disconnect' as event_type, is_wl_cancel_user
    from dwm_yz_tb_comm_cm_all_final
    where par_month_id = ${event_current_month}
      and is_cancel_user = 1
      and prod_type = 40 and kd_desc = '普通宽带'
) t;
```

### 步骤5：打标宽表（核心）

```sql
drop table if exists tmp_{scope}_tagged_wide purge;
create table tmp_{scope}_tagged_wide ... as
select
    u.serv_id, u.acc_nbr, u.snap_flag, u.cur_flag,
    -- 拍照月状态
    sf.subst_name as snap_subst_name, sf.branch_name as snap_branch_name,
    -- 上月状态
    cf.subst_name as cur_subst_name, cf.branch_name as cur_branch_name,
    cf.is_cancel_user as cur_is_cancel, cf.is_cz as cur_is_cz,
    -- 事件日志
    ni.par_month_id as install_month,
    dc.par_month_id as disconnect_month, dc.is_wl_cancel_user,
    -- flow_type
    case
        when u.snap_flag=1 and u.cur_flag=1 then '保持'
        when u.snap_flag=1 and u.cur_flag=0 then '流失'
        when u.snap_flag=0 and u.cur_flag=1 then '流入'
    end as flow_type,
    -- tag_reason（细分原因）
    case
        -- 保持
        when u.snap_flag=1 and u.cur_flag=1
            then '仍在该营服到达'
        -- 流失：跨机构转移优先判断
        when u.snap_flag=1 and u.cur_flag=0
         and cf.serv_id is not null and cf.is_cancel_user=0 and cf.is_cz=1
         and cf.subst_name='${subst_name}' and cf.branch_name<>'${branch_name}'
            then '转去同分公司其他营服'
        when u.snap_flag=1 and u.cur_flag=0
         and cf.serv_id is not null and cf.is_cancel_user=0 and cf.is_cz=1
         and cf.subst_name<>'${subst_name}'
            then '转去其他分公司'
        -- 流失：计转非
        when u.snap_flag=1 and u.cur_flag=0
         and cf.serv_id is not null and cf.is_cancel_user=0 and cf.is_cz=0
            then '计转非（未拆机但不出账）'
        -- 流失：拆机（事件日志）
        when u.snap_flag=1 and u.cur_flag=0 and dc.is_wl_cancel_user=1
            then concat('物理拆机（', cast(dc.par_month_id as string), '月）')
        when u.snap_flag=1 and u.cur_flag=0 and dc.serv_id is not null
            then concat('逻辑拆机（', cast(dc.par_month_id as string), '月）')
        -- 流失：号码不存在
        when u.snap_flag=1 and u.cur_flag=0 and cf.serv_id is null and dc.serv_id is null
            then '号码不存在（可能已销户或月表未覆盖）'
        when u.snap_flag=1 and u.cur_flag=0 then '其他流失'
        -- 流入：跨机构转入优先判断
        when u.snap_flag=0 and u.cur_flag=1
         and sf.serv_id is not null and sf.subst_name='${subst_name}' and sf.branch_name<>'${branch_name}'
            then '从同分公司其他营服转入'
        when u.snap_flag=0 and u.cur_flag=1
         and sf.serv_id is not null and sf.subst_name<>'${subst_name}'
            then '从其他分公司转入'
        -- 流入：新入网（事件日志）
        when u.snap_flag=0 and u.cur_flag=1 and ni.serv_id is not null
            then concat('新入网（', cast(ni.par_month_id as string), '月）')
        -- 流入：拍照月不在网
        when u.snap_flag=0 and u.cur_flag=1 and sf.serv_id is null and ni.serv_id is null
            then '新到达（拍照月不在网或月表未覆盖）'
        when u.snap_flag=0 and u.cur_flag=1 then '其他流入'
    end as tag_reason
from tmp_{scope}_union_serv u
-- 拍照月全量（分公司范围，不限营服）
left join (select serv_id,subst_name,branch_name
           from dwm_yz_tb_comm_cm_all_mon_final
           where par_month_id=${snapshot_month} and subst_name='${subst_name}'
             and prod_type=40 and kd_desc='普通宽带') sf
  on u.serv_id=sf.serv_id
-- 上月全量（分公司范围，不限营服）
left join (select serv_id,subst_name,branch_name,is_cancel_user,is_cz
           from dwm_yz_tb_comm_cm_all_final
           where par_month_id=${current_month} and subst_name='${subst_name}'
             and prod_type=40 and kd_desc='普通宽带') cf
  on u.serv_id=cf.serv_id
-- 入网事件（取最早）
left join (select serv_id,par_month_id,
           row_number() over(partition by serv_id order by par_month_id asc) as rn
           from tmp_{scope}_monthly_event_log where event_type='new_install') ni
  on u.serv_id=ni.serv_id and ni.rn=1
-- 拆机事件（取最近）
left join (select serv_id,par_month_id,is_wl_cancel_user,
           row_number() over(partition by serv_id order by par_month_id desc) as rn
           from tmp_{scope}_monthly_event_log where event_type='disconnect') dc
  on u.serv_id=dc.serv_id and dc.rn=1;
```

### 步骤6：汇总

```sql
drop table if exists tmp_{scope}_flow_summary purge;
create table tmp_{scope}_flow_summary ... as
select flow_type, tag_reason, count(distinct serv_id) as serv_cnt
from tmp_{scope}_tagged_wide
group by flow_type, tag_reason
sort by flow_type, serv_cnt desc;
```

## 自检要点

1. **拍照月到达 = 保持 + 流失**：`sum(snap_flag) = retained + churned`
2. **上月到达 = 保持 + 流入**：`sum(cur_flag) = retained + inflow`
3. **净增 = 流入 − 流失** = `sum(cur_flag) − sum(snap_flag)`
4. **事件日志量级**：各月入网/拆机数是否合理
5. **月表覆盖**：确认月表是否覆盖所需历史月份

## 风险

- **跨分公司转移**：步骤5 全量只扫指定分公司范围；如需精确检测跨分公司转移，需去掉 `subst_name` 过滤扫描全省
- **月表覆盖**：事件日志月表取拍照月到上月之间，需确认月表是否已覆盖
- **当月装当月拆**：同一号码同月 `is_new_user=1` 且 `is_cancel_user=1`，事件日志记录两条；流失分类中拆机优先于跨机构转移
