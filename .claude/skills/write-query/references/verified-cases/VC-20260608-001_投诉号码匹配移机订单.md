# 投诉号码匹配移机订单

来源：`docs/2026第一周sql案例/陈浩南-潘莉莉-投诉号码清单匹配移机.sql`

**定位**：投诉号码清单驱动，先处理移动号到融合套内宽带的转换，再按投诉归档日期前后 30 天匹配移机订单。

## 适用

- 用户给投诉号码清单，要求判断投诉归档日前后 N 天是否有移机成功订单。
- 附件号码可能包含宽带号码和移动号码；移动号码需要按融合套内宽带号查移机。
- 输出移机订单编码、竣工时间、移机前后局向、营服、网格。
- 多条移机订单命中时，按用户确认默认取最接近投诉归档日期的一条。

## 不适用

- 只查普通移机订单明细：直接用 118 移机订单表。
- 普通入网、到达、在网规模：仍用 069 或指标口径。
- 投诉清单本身统计：先看 059 客服投诉抱怨清单或用户附件，不套本流程。

## 主表与口径

- **移机事实表**：118 移机订单表 `dwd_yz_rpt_comm_ba_subs_move_final`。
- **号码关系表**：069 全业务资料月表 `dwm_yz_tb_comm_cm_all_mon_final`。
- **移动号转宽带**：按同账期同 `rh_tc_id` 找套内宽带，条件 `prod_type=40 and is_rh_ykj=1 and coalesce(prod_type2,0)<>50`。
- **时间窗**：默认投诉归档日前后 30 天，即 `投诉归档日期 between date_sub(subs_stat_date,30) and date_add(subs_stat_date,30)`。
- **多单规则**：默认取最接近投诉归档日期的一条；如用户明确要求保留全部订单，需说明输出行数会膨胀。

## 字段映射

| 需求字段 | 来源 |
|----------|------|
| 投诉工单编号 | 用户附件 |
| 投诉号码 / 接入号 | 用户附件 |
| 投诉归档日期 | 用户附件 |
| 投诉月份 | 用户附件 |
| 融合套餐 ID | 069 月表 `rh_tc_id` |
| 套内宽带号 | 069 月表宽带行 `acc_nbr` |
| 最终关联号 | `case when prod_type=30 then rhkd_acc_nbr else 附件.acc_nbr end` |
| 移机订单编码 | 118 `subs_code` |
| 移机订单竣工时间 | 118 `subs_stat_date` |
| 移机前落地局向 / 营服 / 网格 | 118 `std_subst_name_last` / `std_branch_name_last` / `cell_name_last` |
| 移机后落地局向 / 营服 / 网格 | 118 `std_subst_name` / `std_branch_name` / `cell_name` |

## 过程表血缘

| 步骤 | 用途 | 上游 | 关键规则 | 验收 |
|------|------|------|----------|------|
| 1 | 导入投诉附件 | 用户附件 | 保留投诉工单、号码、归档日期、投诉月份 | 附件行数 |
| 2 | 取投诉号码产品类型和融合套餐 | 附件 + 069 月表 | `acc_nbr + ts_month = par_month_id` | 069 命中行数 |
| 3 | 找套内宽带号码 | 步骤 2 + 069 月表 | 同 `rh_tc_id + par_month_id`，宽带且严口径融合 | 转换后行数 |
| 4 | 抽取移机订单 | 118 | 分区覆盖投诉月份前后 30 天 | 移机订单行数 |
| 5 | 时间窗匹配 | 步骤 3 + 步骤 4 | `gl_acc = move.acc_nbr` + 前后 30 天 | 匹配行数 |
| 6 | 多单去重 | 步骤 5 | 按日期差绝对值取最近一条 | 输出行数 |

## SQL 编排要点

- 使用 CTAS 流水线，便于逐步核对附件命中、移动号转宽带和移机订单匹配情况。
- 069 需要用月表，因为投诉月份是历史快照口径。
- 移机表分区只做读数范围控制；最终是否命中必须看 `subs_stat_date` 与投诉归档日期的业务时间窗。
- 取最近订单建议排序：

```sql
row_number() over (
  partition by ts_nbr, acc_nbr, gd_date
  order by abs(datediff(gd_date, subs_stat_date)),
           subs_stat_date desc,
           subs_code desc
) as rn
```

## 风险审计

- 如果移动投诉号码没有命中 `rh_tc_id` 或没有套内宽带号，需要单独输出未匹配原因。
- 如果同一移动号同账期匹配多个套内宽带，先保留过程表核对，不要静默去重。
- 如果用户改口要求“全部保留订单”，删除最近一条 `rn=1` 过滤，并在交付说明中解释一对多膨胀。
- 号码输出默认脱敏；需要明文时由用户确认。
